FHIR JSON Validation Test Suite

Overview

This test suite validates the custom FHIR JSON validator plugin by testing various scenarios including basic validation, implementation guide constraints, profile validation, and error handling.

Test Cases

1. Validate FHIR Patient Resource - Basic

Purpose: Tests basic FHIR Patient resource validation without additional constraints.

Expected Result: SUCCESS

Test Data: patient-basic.json

2. Validate FHIR Patient Resource - With Implementation Guide

Purpose: Tests FHIR Patient validation against a specific implementation guide.

Expected Result: SUCCESS

Test Data: patient-basic.json

Parameters: ig=hl7.fhir.r4.core

3. Validate FHIR Patient Resource - With Profile

Purpose: Tests FHIR Patient validation against a specific profile.

Expected Result: SUCCESS

Test Data: patient-basic.json

Parameters: profile=http://hl7.org/fhir/StructureDefinition/Patient

4. Validate FHIR Patient Resource - With IG and Profile

Purpose: Tests FHIR Patient validation against both implementation guide and profile constraints.

Expected Result: SUCCESS

Test Data: patient-basic.json

Parameters: ig=hl7.fhir.r4.core, profile=http://hl7.org/fhir/StructureDefinition/Patient

5. Validate Invalid JSON

Purpose: Tests the validator's handling of malformed JSON input.

Expected Result: FAILURE

Test Data: invalid-json.json

Note: This file contains a missing comma, making it invalid JSON.

6. Validate Non-FHIR JSON

Purpose: Tests the validator's handling of valid JSON that is not a FHIR resource.

Expected Result: FAILURE

Test Data: non-fhir.json

Note: This file contains valid JSON but lacks the required resourceType field.

Prerequisites

  • FHIR JSON validator plugin deployed to domain fhir-validation
  • ITB instance running and accessible
  • Domain configured with the plugin in config.properties

Usage

Import this test suite into your ITB instance using the Test Suite management interface. The test suite will automatically create the necessary actors and test cases.

Custom Parameters

The validator plugin supports the following custom parameters:

  • ig - Implementation guide identifier (e.g., "hl7.fhir.r4.core")
  • profile - Profile URL for validation (e.g., "http://hl7.org/fhir/StructureDefinition/Patient")

Standard ITB Inputs

The following inputs are automatically provided by ITB:

  • contentToValidate - File path to the content being validated
  • domain - Domain identifier
  • validationType - Type of validation
  • tempFolder - Temporary folder for processing
  • locale - Locale for error messages
